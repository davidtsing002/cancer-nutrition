# 愈健·院后智能营养方案系统 - 项目规范

## 1. 概念与愿景

**愈健**是一个专注于出院后营养管理的智能平台，致力于解决"病人生病后不知道怎么吃"这个核心问题。

不同于医院内的营养指导，我们专注于患者回归居家后的持续营养管理——通过整合疾病诊断、营养状态评估、代谢特点分析和药物-营养相互作用检测，在数秒内生成完全个性化的营养方案，并建立院后闭环管理路径。

**核心理念**：让每一位出院患者，都能获得"专属营养师"般的贴心照护。

## 4. 核心功能

### 4.1 患者信息采集
**输入维度**：
1. **基本信息**：年龄、性别、身高、体重
2. **疾病信息**：诊断、手术类型、当前治疗阶段
3. **营养状态**：食欲、胃肠功能、咀嚼吞咽能力
4. **代谢特点**：血糖、血压、血脂基础数据
5. **用药情况**：处方药物（关键：药物-营养相互作用）
6. **生活方式**：饮食习惯、过敏原、烹饪条件

**输入方式**：
- 结构化表单（快速、精准）
- 语音输入（自然语言描述症状）
- 拍照上传（药物包装、检验报告 OCR）

### 4.2 AI营养方案生成
**核心技术**：
- 深度学习模型：基于数百万临床营养案例训练
- 自然语言处理：理解患者主诉，提取关键信息
- 大数据分析：整合最新营养学研究、食材营养数据库

**生成内容**：
- 个性化食谱（三餐+加餐，含热量、营养素）
- 食材替换建议（适应地域、季节、价格）
- 饮食禁忌清单（基于药物相互作用）
- 烹饪指导（简易步骤，适合居家）
- 注意事项（与药物服用的时间间隔）

**生成速度**：目标 < 10秒

### 4.3 院后闭环管理
```
[出院评估] → [方案生成] → [居家执行] → [随访提醒] → [数据回传] → [方案迭代]
     ↑                                                                 ↓
     ←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←
```

**居家执行**：
- 每日食谱推送
- 食材采购清单（智能关联电商）
- 烹饪计时器
- 进食记录打卡

**随访提醒**：
- 定时提醒（用药、进食、饮水）
- 定期问卷（体重、症状、食欲）
- 异常预警（数据异常时主动干预）

**方案迭代**：
- 基于随访数据自动调整
- 阶段性深度评估
- 里程碑提醒（复查、换药）

### 4.4 智能助手（ChatBot）
**能力**：
- 7x24小时在线答疑
- 食物替换建议
- 症状咨询（何时需要就医）
- 情绪支持（鼓励、提醒）

## 6. 技术方案

### 前端技术栈
- **框架**：React 18 + TypeScript
- **构建**：Vite
- **样式**：Tailwind CSS + CSS Variables
- **组件库**：shadcn/ui
- **图标**：Lucide React
- **动画**：Framer Motion
- **图表**：Recharts

### 状态管理
- React Context（全局状态）
- localStorage（数据持久化）

### AI能力模拟
由于是前端展示，我们模拟以下能力：
- 表单数据 → JSON 结构化
- 本地算法生成"合理"的营养方案
- 基于规则的药物-营养相互作用检测
- 预定义的食材数据库

### 数据结构
```typescript
interface PatientProfile {
  basic: { age, gender, height, weight };
  disease: { diagnosis, surgery, stage };
  nutrition: { appetite, digestion, swallowing };
  metabolism: { bloodSugar, bloodPressure, bloodLipid };
  medications: Medication[];
  lifestyle: { diet, allergies, cooking };
}

interface NutritionPlan {
  dailyPlans: DailyPlan[];
  restrictions: string[];
  recommendations: string[];
  warnings: Warning[];

  date: string;
  meals: Meal[];

  type: 'breakfast' | 'lunch' | 'dinner' | 'snack';

}
```
